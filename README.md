# 궁통보감 사주 AI 데이터 자동 추출 파이프라인

본 프로젝트는 구글 **NotebookLM** 및 통합 CLI 도구(**nlm**)를 활용하여, 한글 인코딩이 깨진 궁통보감 PDF 문서로부터 10천간 × 12개월령에 해당하는 명리학 AI 알고리즘 조건식 데이터를 **100% 완전 자동화하여 정밀 추출**하고 교차 검증하는 파이프라인입니다.

---

## 1. 주요 특징

1. **한글 깨짐 우회**: 로컬 텍스트 추출 라이브러리(`pypdf` 등)의 인코딩 한계를 구글 NotebookLM의 강력한 문서 분석 엔진을 통해 극복합니다.
2. **리소스 최적화 (3개월 분할)**: 120번의 API 호출 대신 계절별(3개월 단위)로 묶어 총 40회 질의를 수행합니다. 
3. **3단계 재귀 팩트체크 (Recursive Fact-Check)**:
   - **1차 (원전 대조)**: 타 천간 내용 및 오행 법칙 혼입 여부 검사.
   - **2차 (명리 논리 검증)**: 용신, 희신, 기신 간 생극제화 및 조후 관계의 학술적 모순 보정.
   - **3차 (구조 스키마 유효성)**: 최종 JSON의 구조적 안정성 확보.
4. **체크포인트(이어받기) 지원**: Rate Limit이나 네트워크 불안정으로 실행이 중단되더라도, 마지막으로 성공한 조합부터 즉시 재개할 수 있습니다 (`checkpoint_extraction.json`).

---

## 2. 사전 준비 사항

자동화 스크립트가 실행 중인 크롬 브라우저 세션을 감지하여 구글 계정으로 연동할 수 있도록 **CDP(Chrome DevTools Protocol)** 모드로 크롬을 1회 실행해 주어야 합니다.

1. 모든 크롬 창을 완전히 종료합니다.
2. 윈도우 실행 창(`Win + R`)을 열고 아래 명령을 붙여넣어 실행합니다:
   ```cmd
   "C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir="C:\Users\tslee\chrome_debug_profile"
   ```
3. 켜진 크롬 창에서 [Google NotebookLM](https://notebooklm.google.com/)에 접속하여 로그인을 마칩니다.
4. 이 상태에서 파이프라인 스크립트를 가동하면 세션을 자동으로 가져와 가동됩니다.

---

## 3. 실행 방법

### 의존성 패키지 확인
파이프라인 실행 전 필요한 패키지들이 설치되어 있는지 확인합니다:
```bash
pip install notebooklm-mcp notebooklm-mcp-cli websocket-client
```

### 단일 조합 테스트 가동 (1개 조합 -> 3개 행 추출)
을목(乙木)의 봄(인, 묘, 진월) 조합 하나만을 대상으로 전체 파이프라인(노트북 확인, PDF 업로드, 3단계 재귀검증, 파싱)이 안전하게 도는지를 검증합니다.
```bash
python notebooklm_extractor.py --test
```

### 전체 40회 실전 가동 (10천간 x 4계절 -> 120개 행 완성)
10천간과 4계절에 대해 전체 데이터를 추출하고 최종 데이터베이스 파일을 빌드합니다.
```bash
python notebooklm_extractor.py
```

## 4. 최종 출력물

추출이 완료되면 `docs` 디렉토리 하위에 두 종류의 데이터베이스가 생성됩니다.

1. **`docs/gungtong_database.json`**
   - 120개 월령별 데이터가 용신 정보 및 조건식 리스트를 포함한 계층형 중첩 구조(Nested JSON)로 저장되어 명리학 관계를 완벽하게 보존합니다.
2. **`docs/gungtong_database.csv`**
   - 관계형 DB 적재나 엑셀 분석이 용이하도록 조건식별로 평탄화(Flatten)한 2차원 테이블 형태로 저장됩니다.
   - **CSV 컬럼**: `일간`, `계절`, `월별`, `핵심용신`, `보조용신`, `취용요약`, `조건`, `결과`
